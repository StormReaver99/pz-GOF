# pz-GOF — Патерни проєктування GoF

Практична робота з реалізації патернів проєктування GoF.
Тема: система університету. Мова: TypeScript.

## Реалізовані патерни

### Породжувальні
- **Singleton** — реєстр університету в одному екземплярі
- **Factory Method** — створення лекційних та лабораторних курсів
- **Builder** — покрокове складання розкладу занять

### Структурні
- **Adapter** — конвертація оцінок між старою (1-5) та новою (A-F) системою
- **Facade** — вступна кампанія за одним викликом замість 3 підсистем
- **Decorator** — додавання онлайн-трансляції та тестування до курсу

### Поведінкові
- **Observer** — студенти підписуються на сповіщення від деканату
- **Strategy** — алгоритм підрахунку GPA змінюється без зміни класу
- **Command** — відрахування та поновлення студентів зі скасуванням

## Запуск

npm install
npm start

## Висновки

Патерни GoF допомагають структурувати код так щоб його було легко
розширювати. Наприклад Factory Method дозволяє додати новий тип курсу
без зміни існуючого коду, а Command дозволяє скасовувати дії.
Без патернів той самий функціонал перетворюється на великий клубок
умов та залежностей.
