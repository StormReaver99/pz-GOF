# Практичне заняття: Реалізація патернів проєктування (GoF)

Це моя практична робота (pz-GOF), в якій я реалізував 8 класичних патернів проєктування мовою TypeScript. 

Щоб не робити стандартні і нудні приклади з фігурами чи автомобілями з підручників, я вирішив застосувати патерни до того, що мені ближче — механік відеоігор (на прикладі багатокористувацьких шутерів). Це допомогло мені краще зрозуміти, як саме вони працюють на практиці.

## Які патерни я реалізував і навіщо:

### 1. Породжувальні (Creational)
* **Singleton (`GameSettings.ts`)**: Використав для загальних налаштувань гри. Налаштування мають бути одні на весь клієнт, тому цей патерн підходить ідеально — він просто не дає створити кілька копій класу.
* **Factory Method (`WeaponFactory.ts`)**: Зробив фабрику для видачі зброї (штурмової чи снайперської). Дуже зручно: якщо треба буде додати нову зброю, основний код міняти не доведеться.
* **Builder (`LoadoutBuilder.ts`)**: Використав для збірки екіпіровки гравця в лобі. Дозволяє гнучко "збирати" клас бійця по шматочках (зброя, перки тощо), а не передавати купу параметрів в один конструктор.

### 2. Структурні (Structural)
* **Adapter (`InputAdapter.ts`)**: Написав адаптер, який дозволяє грати на ПК з геймпада. Він бере старий інтерфейс геймпада і "перекладає" його команди в зрозумілі для ПК.
* **Facade (`MatchmakingFacade.ts`)**: Сховав під одну просту кнопку "Грати" купу складних процесів (авторизація на сервері, пошук лобі, завантаження карти).
* **Decorator (`WeaponDecorator.ts`)**: Крута штука для обвісів на зброю. Замість того, щоб створювати купу нових підкласів для зброї з глушником чи коліматором, я просто динамічно "обгортаю" базову зброю потрібними модулями.

### 3. Поведінкові (Behavioral)
* **Strategy (`StanceStrategy.ts`)**: Дозволяє міняти стиль руху бійця (спринт або лежачи) прямо під час гри. Це набагато краще і чистіше, ніж писати купу `if/else` в одному класі гравця.
* **Observer (`Killfeed.ts`)**: Зробив ігровий кілфід (журнал вбивств). Сервер виступає видавцем, а всі гравці — підписниками. Коли хтось когось ліквідує, сервер розсилає повідомлення всім гравцям автоматично, ніхто нікого постійно не опитує.

## Як це все запустити

Для початку треба встановити залежності проєкту:
npm install
Потім просто запускаємо головний демонстраційний файл:
npm start
У консоль виведеться покроковий результат роботи кожного з восьми патернів.

## Мої висновки
Під час виконання цієї роботи я зрозумів, що патерни GoF — це не просто суха теорія, а реально корисний інструмент. Спочатку здається, що писати стільки класів та інтерфейсів — це оверінжиніринг і зайва трата часу. Але коли уявити, що система почне розростатися, стає зрозуміло, що це рятує від купи багів. 

Код вийшов розбитим на логічні блоки. Наприклад, завдяки Декоратору чи Фабричному методу я можу додати новий тип гранати чи новий обвіс взагалі без переписування старого коду . Архітектура стала набагато гнучкішою.
